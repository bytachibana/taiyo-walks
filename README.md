# Taiyo's Danville Walk Tracker

A self-updating map of every road Taiyo has walked in Danville, CA. Export a
GPX from Runkeeper, drop it in a Google Drive folder, and within ~15 minutes
the map at **taiyo.spring-rock.com** updates itself. No other manual steps.

## How it works

```
Runkeeper GPX  ->  Google Drive folder
                        |  Apps Script (every 15 min): pushes new GPX -> GitHub walks/
                        v
                   GitHub repo (taiyo-walks)
                        |  GitHub Actions (on push to walks/): runs process_walks.py,
                        |  commits progress.json back
                        v
                   GitHub Pages  ->  taiyo.spring-rock.com
```

- **`process_walks.py`** ‚Äî reads every GPX in `walks/`, buffers each track by
  15 m, clips it against `roads.geojson` (Shapely), writes `progress.json`.
- **`index.html`** ‚Äî Leaflet map; fetches `progress.json` and renders walked
  (green) vs. unwalked (gray) roads plus GPS tracks.
- **`roads.geojson`** ‚Äî static Danville road network (4,239 segments, ~1.85 MB).
  Never regenerated by the pipeline.
- **`progress.json`** ‚Äî the only file the pipeline changes.

## One-time setup

### 1. GitHub repo
1. Create a repo named **`taiyo-walks`**.
2. Commit everything, including `walks/*.gpx`, `roads.geojson`, `index.html`,
   and `CNAME`. (`roads.geojson` is 1.85 MB ‚Äî commit it directly, **no Git LFS**.)

### 2. GitHub Pages
1. Settings ‚Üí Pages ‚Üí **Source: Deploy from a branch**, Branch: `main`, `/ (root)`.
2. Custom domain: `taiyo.spring-rock.com` (the committed `CNAME` file declares it).
3. Leave "Enforce HTTPS" on once the cert provisions.

### 3. DNS (GoDaddy, for spring-rock.com)
Add a CNAME record:

| Type  | Name  | Value                     | TTL    |
|-------|-------|---------------------------|--------|
| CNAME | taiyo | `bytachibana.github.io`   | 1 hour |

This only affects the `taiyo` subdomain; `www.spring-rock.com` is untouched.

### 4. GitHub token for Apps Script
Create a **fine-grained Personal Access Token** (github.com ‚Üí Settings ‚Üí
Developer settings ‚Üí Personal access tokens ‚Üí Fine-grained tokens):
- Repository access: **only `taiyo-walks`**
- Permissions: **Contents ‚Üí Read and write**

This is the only external token. The Actions workflow's commit-back uses the
built-in `GITHUB_TOKEN` and needs no secret.

### 5. Google Apps Script
1. Create a standalone Apps Script project; paste `apps_script/DriveWatcher.gs`.
2. Project Settings ‚Üí Script Properties:
   - `GH_PAT` = the fine-grained PAT from step 4
   - `REPO_OWNER` = `bytachibana`
   - `REPO_NAME` = `taiyo-walks`
   - `DRIVE_FOLDER_ID` = `1TCjKm48cSIpfjrnJ0ECrtb90rO961jez`
3. Run `watchDriveFolder()` once from the editor to authorize scopes; check the
   execution log for errors.
4. Add a trigger: **Triggers ‚Üí Add Trigger ‚Üí `watchDriveFolder`, time-driven,
   minute timer, every 15 minutes.**

## Ongoing use

1. Export GPX from Runkeeper.
2. Save it to the Google Drive folder.
3. Wait ~15 min (Apps Script) + ~2 min (Actions).
4. Visit **taiyo.spring-rock.com**.

## Local run (optional)

```bash
pip install shapely
python process_walks.py      # reads walks/*.gpx + roads.geojson -> progress.json
```

Then open `index.html` in a browser.

## Testing the pipeline

1. Upload a test GPX to `walks/` via the GitHub web UI.
2. Actions tab ‚Üí the **Process Walks** workflow should start within seconds.
3. After ~90 s, confirm a new commit updated `progress.json`.
4. Check taiyo.spring-rock.com.

## Privacy: no time-of-day is published

Runkeeper GPX files embed the exact time of every track point and the walk
time in the file name. Because the repo (and the map) are public, all
time-of-day data is stripped before anything is published:

- **File contents** ‚Äî every `<time>` is zeroed to `T00:00:00Z` (date kept) and
  the `<name>` clock time is removed. Done for existing files, and by the Apps
  Script for every future upload.
- **File names** ‚Äî GPX files are stored as `YYYY-MM-DD.gpx`, or
  `YYYY-MM-DD-1.gpx`, `-2.gpx` ... for multiple walks on the same day.
- **Map labels** ‚Äî the sidebar shows the date only, never a clock time.

The **date** is intentionally retained (it drives the walk list). Coordinates
are unchanged.

## Notes

- `process_walks.py` reprocesses **all** GPX files every run (intentional ‚Äî
  simple, and fast at this scale).
- 15 m buffer ‚âà sidewalk-level accuracy. Motorways are excluded.
- The Actions trigger is path-filtered to `walks/**`, and `GITHUB_TOKEN`
  commits don't retrigger workflows, so the commit-back cannot loop.
